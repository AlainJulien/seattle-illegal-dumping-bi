\# Contributing Guidelines



Thank you for your interest in contributing! Any insight for improvement will be appreciated!



\## How to Contribute



\### Reporting Bugs

\- Use the GitHub Issues bug report template

\- Include Python version, OS, and error message

\- Attach sample data if possible (anonymized)



\### Suggesting Features

\- Use the GitHub Issues feature request template

\- Explain the business problem it solves

\- Include mockups or examples if applicable



\### Submitting Code

1\. Fork the repository

2\. Create a branch: `git checkout -b feature/your-feature-name`

3\. Make your changes

4\. Run tests: `pytest scripts/tests/`

5\. Format code: `black scripts/`

6\. Submit a pull request



\## Code Style

\- Python: PEP 8, formatted with black

\- DAX: Use variables, always handle division by zero

\- SQL: Uppercase keywords, snake\_case table/column names

\- Documentation: Clear, concise, include examples



\## Contribution Areas

\- New DAX measures or optimizations

\- Additional SQL analysis scripts

\- Dashboard page improvements

\- Test coverage expansion

\- Documentation improvements
